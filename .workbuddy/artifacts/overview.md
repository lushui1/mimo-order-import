# 第九轮修复概述

## 问题
1. AI 解析时间太长（mimo-v2.5 是 reasoning 模型，96% token 浪费在内部推理）
2. 历史页运单记录没有聚合展示

## 修复

### AI 策略重构：审核模式
| 改动 | 之前 | 之后 |
|------|------|------|
| 策略 | AI 从零生成规则 | **本地分析 + AI 审核补全** |
| System Prompt | 1500 字 | **<300 字** |
| 文件预览 | 2500 字符 | **600 字符** |
| AI 输入 | 只有文件内容 | **文件内容 + 本地分析结果** |
| AI 任务 | 找表头 + 映射列 + 检测结构 | **只需修正本地结果** |
| 超时 | 55s | **30s** |

**关键发现**: mimo-v2.5 是 reasoning 模型，测试中 96% 的 token 用在内部推理，只有 4% 留给实际输出。通过传入本地分析结果，AI 只需做"修正"而非"生成"，大幅减少推理工作量。

### 历史页聚合展示
- 按外部编码分组为出库单卡片
- 同一外部编码下的 SKU 合在一起展示
- 收货信息（收件人/电话/地址）共享展示
- 可展开/折叠查看 SKU 明细表
- 统计：X 个出库单 · Y 条 SKU 记录
